# 🏛️ Death Registration System (DRS)

![PHP Version](https://img.shields.io/badge/PHP-8.1%2B-7A86B8?logo=php)
![Architecture](https://img.shields.io/badge/Architecture-Custom_MVC-blue)
![Database](https://img.shields.io/badge/Database-MySQL%20%7C%20PDO-orange)
![Security](https://img.shields.io/badge/Security-OWASP_Compliant-success)

A modern, enterprise-grade **Civil Registration Web Application** built natively in PHP. This platform handles the digital extraction, approval, and certification of death records with a focus on strict Role-Based Access Control (RBAC), robust security, and real-time interactive analytics.

## ✨ Key Features

* **Custom MVC Architecture:** Zero bloat. Built from the ground up using strict OOP PHP principles, an automated autoloader, and a centralized routing engine.
* **Role-Based Access Control (RBAC):** Strict separation of concerns between `Admins` (system management, global analytics, record approvals) and `Clerks` (record entry, localized metrics).
* **Official e-Certificates:** Automatically generates highly realistic, print-ready Official Certificates of Death featuring digital signatures and dynamic validation QR codes.
* **Real-Time Analytics Dashboard:** Leveraging `Chart.js` to provide filterable, dynamic insights including regional breakdowns, status tracking, and cause-of-death aggregates.
* **Enterprise Security:**
  * Strict PDO Prepared Statements (SQLi Immunity).
  * Enforced CSRF tokenization on all mutable endpoints.
  * Deep output sanitization (`htmlspecialchars`) and input validation engines.
  * Production-hardened session configuration and Content Security Policies.
* **Security Audit Logs:** A comprehensive tracking system monitoring all user authentication events, record mutations, and system changes for absolute accountability.
* **Context-Aware Exports:** Instantly export filtered datasets natively to UTF-8 BOM formatted CSVs for seamless Microsoft Excel integration.

## 🚀 Getting Started

### Prerequisites
* PHP 8.1 or higher
* MySQL / MariaDB Server
* A local web server (Apache, Nginx, XAMPP, or the built-in PHP server)

### 1. Installation

Clone the repository to your local development environment:
```bash
git clone https://github.com/yourusername/death-registration-system.git
cd death-registration-system
```

### 2. Database Configuration

1. Create a new MySQL database named `death_registration`.
2. Import the provided schema (if available) or rely on the core setup.
3. The system is designed for production. Configure your environment variables on your server or update your local setup:
   - `DB_HOST` (default: 127.0.0.1)
   - `DB_PORT` (default: 3306)
   - `DB_NAME` (default: death_registration)
   - `DB_USER` (default: root)
   - `DB_PASS` (default: empty)

### 3. Running the Application

**Option A: Built-in PHP Server (Fastest)**
Run this command from the root of the project:
```bash
php -S localhost:8000 index.php
```
Then visit `http://localhost:8000` in your browser.

**Option B: Traditional Server**
Point your Apache/Nginx `DocumentRoot` directly to the project root directory. Ensure all traffic is routed through `index.php`.

### Default Admin Credentials
* **Username:** `admin`
* **Password:** `Admin@123`
> *Note: It is strictly recommended to change these credentials immediately after your first login.*

## 🛡️ Security Overview

This application was engineered with a "Zero-Trust" mindset:
- **No Direct File Access:** All web traffic is securely funneled through `index.php`.
- **Environment Variables:** Credentials are dynamically injected, keeping sensitive data out of the source code.
- **Flash Notifications:** Custom DOM-manipulation logic securely manages transient UI states without exposing persistent data.
